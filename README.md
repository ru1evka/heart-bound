# Сердце в переплёте — сайт любовных романов

## Структура проекта

```
siteboock/
├── server.js           — Node.js сервер (Express + SQLite)
├── package.json        — зависимости
├── data.db             — база данных SQLite (создаётся автоматически)
├── index.html          — главная страница
├── style.css           — стили сайта
├── script.js           — логика фронтенда (загрузка из API)
├── admin.html          — админ-панель
├── admin.css           — стили админки
├── admin.js            — логика админки (CRUD через API)
└── assets/
    ├── favicon.svg
    ├── book1.jpg.svg   — анимированная обложка
    ├── book2.jpg.svg
    ├── book3.jpg.svg
    └── book4.jpg.svg
```

## Как запустить

```bash
# 1. Установить зависимости (один раз)
npm install

# 2. Запустить сервер
npm start
```

Сайт будет доступен по адресу: **http://localhost:3000**
Админ-панель: **http://localhost:3000/admin.html**

## Авторизация

Админ-панель защищена логином и паролем. По умолчанию:
- **Логин:** `admin`
- **Пароль:** `romance2026`

Чтобы изменить, задайте переменные окружения перед запуском:
```bash
set ADMIN_LOGIN=мой_логин
set ADMIN_PASSWORD=мой_пароль
npm start
```

Или отредактируйте значения прямо в `server.js` (строки `ADMIN_LOGIN` и `ADMIN_PASSWORD`).

Без авторизации:
- Главная страница и чтение книг/блога доступны всем
- Админ-панель показывает страницу входа
- API на запись (POST/PUT/DELETE) возвращает 401

## База данных

Используется SQLite (через sql.js — без нативных зависимостей).
Файл `data.db` создаётся автоматически при первом запуске и заполняется дефолтными данными.

Все изменения через админку сохраняются в БД и видны всем посетителям сайта.

## API

### Книги
- `GET /api/books` — список всех книг
- `GET /api/books/:id` — одна книга
- `POST /api/books` — создать книгу
- `PUT /api/books/:id` — обновить книгу
- `DELETE /api/books/:id` — удалить книгу

### Блог
- `GET /api/posts` — список всех записей
- `GET /api/posts/:id` — одна запись
- `POST /api/posts` — создать запись
- `PUT /api/posts/:id` — обновить запись
- `DELETE /api/posts/:id` — удалить запись

## Как добавить обложки

В админке в поле «Обложка» укажите путь к файлу:
- `assets/my-book.mp4` — видео (автоплей, зацикленное)
- `assets/my-book.gif` — анимированный GIF
- `assets/my-book.jpg.svg` — анимированный SVG
- Любой URL на изображение

## Настройка

- **Порт**: переменная окружения `PORT` (по умолчанию 3000)
- **QR-код Telegram**: в `index.html` замените ссылку `https://t.me/` на свою
- **Соцсети**: в `index.html` замените `href="#"` на реальные ссылки

## Особенности

- Данные хранятся в SQLite — переживают перезапуск сервера
- Админка работает через REST API
- Клик на «Читать» в блоге открывает запись в модальном окне (не перекидывает наверх)
- Полностью адаптивный дизайн
- Анимированные обложки (SVG/mp4/gif)
- Без тяжёлых зависимостей — только Express и sql.js
